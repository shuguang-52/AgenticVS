# AgenticVS

AgenticVS is a Video Summarization Agentic Workflow powered by Multimodal Large Language Models (MLLMs) and multimodal features. This project provides a complete pipeline consisting of feature extraction, feature fusion, LLM verification (Verifier), and semantic refinement (Reflector) to achieve high-quality video score prediction and feature training.

---

## 📂 Project Modules Overview

The project consists of the following four core modules:

### 1. Feature Extraction (VideoMAEv2)

Responsible for extracting high-quality Video Embeddings from raw video files.

* **Main Script**: `extract_tad_feature_summe_8frame.py`
* **Description**: Extracted video features default to a window width of **8 frames**.
* **🛠️ Modification Guide**: To change the window width to 16 frames, directly modify lines **97** and **98** in the code.

### 2. Summerizer (Feature Fusion & Mapping)

Responsible for fusing features from different backbones and performing dimensionality reduction.

* **Main Script**: `Summerizer/mae2google_mlp.py`
* **Description**: Fuses features extracted by VideoMAE and GoogleNet, applies an MLP network for dimensionality reduction and mapping, and finally generates the feature data required to train the CSTA model.

### 3. Verifier (Initial Score Verification)

Introduces a Large Language Model (Qwen) to verify the logic and accuracy of the initial scores generated by the Summerizer.

* **Main Scripts**:
* `Verifier/qwen_judge_csta.py`
* `Verifier/qwen_validator.py`


* **Description**: Automatically evaluates and judges whether the `initial score` output by the Summerizer contains deviations or errors, and flags anomalous frames that require modification.

### 4. Reflector (Semantic Re-evaluation & Correction)

Utilizes Vision-Language Models (VLMs) for fine-grained calibration of final scores.

* **Main Script**: `Reflector/qwen_clip_frame_semantic.py`
* **Description**:
1. Uses Qwen to generate semantic descriptions (captions) for individual video frames.
2. Uses the CLIP model to recompute matching scores between text and images.
3. **Efficiency Optimization**: To boost runtime performance, the system *only* recalculates scores for the "frames needing modification" flagged by the Verifier.
4. Replaces the inaccurate initial scores with the corrected scores to produce high-quality final training targets.

## 🚀 Training & Evaluation Guide

Handles the training of the Summerizer model, metric evaluation, and results visualization.

### Core Scripts

* **Model Training**: `train.py`
* **Metric Evaluation**: `inference.py`
* **Results Visualization**: `inference_visual.py`

### 🛠️ Runtime Configuration Changes

Before running the scripts above, please ensure you update the relevant path parameters to match your local environment:

**Training Phase (`train.py`):**

* Required changes:
* **Model Save Path** (Destination for the trained model checkpoints)
* **Feature Vector Read Path** (Ensure the input format is an `.h5` file)


* *Note: Aside from these two path adjustments, leave all other training hyperparameters and code logic untouched.*

**Evaluation Phase (`inference.py`):**

* Required changes:
* **Model Load Path** (Specify the path to your newly trained model weight file)

## 📝 Recommended Execution Workflow

1. Run `extract_tad_feature_summe_8frame.py` to extract basic video features.
2. Run `Summerizer/mae2google_mlp.py` to generate CSTA training features.
3. (Optional) Execute the **Verifier** and **Reflector** modules to obtain the calibrated final score files.
4. Modify paths in `train.py` and start model training.
5. Modify paths in `inference.py` to run model evaluation.
6. Use `inference_visual.py` to inspect the visualized results.
